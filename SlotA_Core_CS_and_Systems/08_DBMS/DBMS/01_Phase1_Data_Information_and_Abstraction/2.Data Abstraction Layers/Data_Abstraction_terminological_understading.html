📘 COMPLETE TERMINOLOGICAL UNDERSTANDING

Data Abstraction + ANSI/SPARC + Schema vs Instance

1. Database (DB)

Meaning

An organized collection of related data stored electronically.

Purpose

  • Store data
  • Retrieve data
  • Modify data
  • Manage information efficiently

Example

Customers
Accounts
Transactions
Loans

2. DBMS (Database Management System)

Meaning

Software that manages databases.

Responsibilities

  • Data storage
  • Query processing
  • Security
  • Concurrency control
  • Backup and recovery
  • Integrity enforcement

Examples

  • MySQL
  • PostgreSQL
  • Oracle Database
  • MongoDB

3. Data Abstraction

Meaning

Technique of hiding unnecessary internal complexity while exposing essential information.

User Sees

  • SQL
  • Tables
  • Views

User Does NOT See

  • Disk sectors
  • Memory pages
  • Buffer algorithms
  • Storage offsets

Purpose

  • Simplicity
  • Security
  • Flexibility
  • Maintainability

4. ANSI/SPARC Architecture

Meaning

Formal 3-level DBMS architecture separating:

  • User views
  • Logical structure
  • Physical storage

Goal

Changes in one level should not affect others.

5. External Level (View Level)

Meaning

Highest abstraction layer visible to users.

Contains

  • Views
  • SQL queries
  • User-specific representations

Focus

“What data should THIS user see?”

Characteristics

  • Multiple external views possible
  • Security-oriented
  • Simplified representation

6. External Schema / View Schema / Subschema

Meaning

Structure of a specific user view.

Example

CREATE VIEW HR_View AS
SELECT Name, Salary FROM Employee;

7. View

Meaning

Virtual table generated from query results.

Purpose

  • Security
  • Simplification
  • Data hiding
  • Abstraction

Types

Type Meaning
Virtual View Generated dynamically
Materialized View Physically stored result

8. Conceptual Level (Logical Level)

Meaning

Middle abstraction layer describing complete logical database structure.

Focus

“What data exists and how does it relate?”

Contains

  • Tables
  • Relationships
  • Constraints
  • Business rules

9. Conceptual Schema / Logical Schema

Meaning

Complete blueprint of database.

Defines

  • Entities
  • Attributes
  • Relationships
  • Constraints
  • Domains

10. Internal Level (Physical Level)

Meaning

Lowest abstraction layer describing physical storage details.

Focus

“How is data physically stored?”

Contains

  • Files
  • Pages
  • Blocks
  • Indexes
  • Compression
  • Encryption

11. Physical Schema / Storage Schema

Meaning

Specification of physical storage organization.

Includes

  • File structures
  • Access paths
  • Index structures
  • Page layouts

12. Data Independence

Meaning

Ability to modify one schema level without affecting higher levels.

13. Physical Data Independence

Meaning

Change physical storage without affecting logical schema.

Examples

  • HDD → SSD
  • B+ Tree → Hash Index
  • Compression enabled

14. Logical Data Independence

Meaning

Change logical schema without affecting external views.

Examples

  • Add columns
  • Split tables
  • Add relationships

15. Schema

Meaning

Blueprint/definition of database structure.

Defines

  • Tables
  • Attributes
  • Constraints
  • Data types
  • Relationships

Nature

  • Static
  • Structural
  • Declarative

16. Instance (Database State)

Meaning

Actual data stored in database at a specific time.

Nature

  • Dynamic
  • Time-dependent
  • Changes frequently

17. Intension

Meaning

Definition/rules of database.

Equivalent To

Schema

18. Extension

Meaning

Actual set of stored tuples.

Equivalent To

Instance

19. Relation

Meaning

Table in relational model.

Components

  • Tuples (rows)
  • Attributes (columns)

20. Relation Schema

Meaning

Structure/definition of relation.

Example

Student(ID:int, Name:varchar)

21. Relation Instance

Meaning

Current set of tuples inside relation.

22. Tuple

Meaning

Single row in relation.

Example

(101, "Alice", "CS")

23. Attribute

Meaning

Column/property of relation.

Example

Student.Name

24. Domain

Meaning

Set of valid values allowed for an attribute.

Example

Age INT CHECK(Age >= 18)

25. Constraint

Meaning

Rule enforced by DBMS to maintain correctness and integrity.

26. Integrity Constraint

Meaning

Rule ensuring database remains valid and consistent.

27. Entity

Meaning

Real-world object represented in database.

Examples

  • Student
  • Employee
  • Product

28. Relationship

Meaning

Association between entities/tables.

Example

Order.Customer_ID → Customer.ID

29. Primary Key

Meaning

Attribute uniquely identifying each tuple.

Properties

  • Unique
  • Not NULL

30. Foreign Key

Meaning

Attribute referencing primary key of another relation.

Purpose

Maintain referential integrity.

31. Referential Integrity

Meaning

Foreign key must reference existing tuple.

32. Candidate Key

Meaning

Any attribute set capable of uniquely identifying tuples.

33. Super Key

Meaning

Any attribute combination uniquely identifying tuples.

34. Composite Key

Meaning

Primary key made of multiple attributes.

35. UNIQUE Constraint

Meaning

Prevents duplicate values.

36. NOT NULL Constraint

Meaning

Attribute cannot contain NULL.

37. CHECK Constraint

Meaning

Validates conditions before insertion/update.

Example

CHECK (Salary > 0)

38. Query

Meaning

Request for data or operations in DBMS.

Example

SELECT * FROM Student;

39. SQL (Structured Query Language)

Meaning

Language used to communicate with relational DBMS.

Categories

Type Purpose
DDL Schema definition
DML Data manipulation
DCL Access control
TCL Transaction control

40. DDL (Data Definition Language)

Meaning

Commands defining database structure.

Examples

CREATE
ALTER
DROP

41. DML (Data Manipulation Language)

Meaning

Commands manipulating data.

Examples

INSERT
UPDATE
DELETE
SELECT

42. Query Optimizer

Meaning

Component choosing most efficient execution plan.

Goal

Minimize:

  • CPU cost
  • Disk I/O
  • Memory usage

43. Execution Plan

Meaning

Strategy chosen by optimizer to execute query.

44. Sequential Scan (Full Table Scan)

Meaning

Reading every row in table.

45. Index Scan

Meaning

Using index structure for faster retrieval.

46. Index

Meaning

Auxiliary data structure accelerating searches.

Examples

  • B+ Tree
  • Hash Index
  • Bitmap Index

47. B+ Tree

Meaning

Balanced tree indexing structure optimized for disk systems.

Features

  • Sorted keys
  • Fast range queries
  • Logarithmic search

48. Hash Index

Meaning

Index using hash function for exact-match lookups.

49. Buffer Manager

Meaning

Component managing movement of pages between RAM and disk.

50. Buffer Pool

Meaning

Memory cache storing disk pages temporarily.

51. File Manager

Meaning

Handles reading/writing raw bytes to disk.

52. Storage Manager

Meaning

Maps logical tables to physical storage.

53. Transaction

Meaning

Logical unit of work executed atomically.

54. Transaction Manager

Meaning

Controls transactions and maintains consistency.

55. ACID Properties

Property Meaning
Atomicity All-or-nothing execution
Consistency Rules preserved
Isolation Transactions independent
Durability Changes survive crashes

56. Concurrency Control

Meaning

Managing simultaneous transactions safely.

57. Locking

Meaning

Restricting concurrent access to data.

58. Recovery System

Meaning

Mechanism restoring database after failures.

59. Query Parsing

Meaning

Analyzing SQL syntax and semantics.

60. Query Execution Engine

Meaning

Component executing optimized query plans.

61. Data Block / Page

Meaning

Fixed-size unit of disk I/O.

Typical Size

4KB–16KB

62. File Organization

Meaning

Method of arranging records physically.

Types

  • Heap File
  • Sorted File
  • Hash File

63. Compression

Meaning

Reducing storage space usage.

64. Encryption

Meaning

Protecting stored data from unauthorized access.

65. Tablespace

Meaning

Logical storage unit mapping data to physical files.

66. RAID

Meaning

Multiple-disk organization for performance/reliability.

67. Normalization

Meaning

Process of reducing redundancy and anomalies.

68. Redundancy

Meaning

Duplicate unnecessary data storage.

69. Data Integrity

Meaning

Correctness and consistency of database data.

70. Separation of Concerns

Meaning

Each DBMS layer has one responsibility.

71. Mapping

Meaning

Translation between abstraction levels.

72. View Updatability

Meaning

Whether DBMS can update underlying tables through a view.

73. ORM (Object Relational Mapping)

Meaning

Technique mapping objects to relational tables.

Examples

  • Hibernate
  • SQLAlchemy

74. Impedance Mismatch

Meaning

Conflict between object-oriented models and relational models.

75. NoSQL

Meaning

Non-relational database systems.

Examples

  • Document DB
  • Graph DB
  • Key-value DB

76. NewSQL

Meaning

Modern systems combining:

  • SQL abstraction
  • Distributed scalability

Examples

  • CockroachDB
  • YugabyteDB

77. Cloud Database

Meaning

Database managed over cloud infrastructure.

Examples

  • Amazon RDS
  • Google BigQuery

78. Inference Attack

Meaning

Deriving hidden information indirectly from accessible data.

79. Row-Level Security

Meaning

Restricting access to specific rows based on user identity.

80. Computed Column

Meaning

Derived value calculated dynamically.

81. Aggregation

Meaning

Summarizing data using functions:

  • SUM
  • AVG
  • COUNT
  • MAX
  • MIN
