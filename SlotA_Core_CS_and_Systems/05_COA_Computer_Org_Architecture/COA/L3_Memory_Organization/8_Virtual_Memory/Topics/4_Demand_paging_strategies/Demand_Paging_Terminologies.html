Section 1: Unified Memory Architecture (Page 1)

Terminology Explanation
Unified Memory Architecture (UMA) A memory design where CPU and GPU share the same physical memory pool, rather than having separate dedicated memories
Demand Paging A memory management technique where pages are loaded into RAM only when needed (on demand), not all at once
Shared Memory Memory space that can be accessed by multiple processors or components (CPU/GPU)

Section 2: Modern Optimizations (Page 1)

Terminology Explanation
TLB (Translation Lookaside Buffer) A hardware cache that stores recent virtual-to-physical address translations to speed up memory access
Page Table Walk The process of traversing the page table hierarchy to find the physical address for a virtual address
Prefetching A technique where the OS predicts which pages will be needed next and loads them into RAM before they’re requested
Huge Pages Larger memory page sizes (2MB or 1GB) used instead of standard 4KB pages to reduce TLB misses for large datasets
Swap Compression (zswap/zram) A technique that compresses pages in RAM instead of writing them to disk, making swap operations faster than disk I/O

Section 3: Worked Example — Page Replacement (Pages 1-2)

Terminology Explanation
Working Set The set of pages a program is actively using at any given time
Page Fault An exception raised when a program accesses a page that is not currently loaded in RAM
Hit Rate The percentage of memory accesses that find the requested page already in RAM (no page fault)
Eviction The process of removing a page from RAM to make space for a new page
Thrashing A state where the system spends most of its time handling page faults rather than executing useful work

Section 4: Key Definitions (Page 2)

Terminology Explanation
Page A fixed-size block of virtual memory (typically 4KB), the basic unit of transfer between disk and RAM
Frame A page-sized slot in physical RAM that can hold one page
Page Table A per-process data structure that maps virtual page numbers to physical frame numbers or disk locations
Page Fault A hardware exception triggered when the CPU accesses a page not present in RAM

Section 5: Demand Paging vs. Load Everything (Page 3)

Terminology Explanation
Virtual Memory A memory management technique that allows programs to use more memory than physically available RAM by using disk as backup
Backing Store / Disk Secondary storage (hard disk/SSD) where pages not in RAM are kept
Lazy Loading Loading data only when it’s actually needed, rather than upfront

Section 6: Locality Principles (Page 4)

Terminology Explanation
Spatial Locality The tendency to access memory locations near recently accessed locations (e.g., sequential array access)
Temporal Locality The tendency to access the same memory locations repeatedly within a short time period
Contiguous Memory Memory blocks that are adjacent to each other in address space
Thrashing When a system has excessive page faults, spending more time swapping pages than executing instructions

Section 7: Program Loading & Startup (Pages 5, 7)

Terminology Explanation
Entry Point The first instruction of a program where execution begins
Startup Time The time from user action (e.g., double-click) to program responsiveness
Disk Bandwidth The rate at which data can be read from disk (measured in MB/s)
Lazy Initialization Delaying the loading/initialization of components until they’re actually needed

Section 8: Memory Access & Page Fault Handling (Pages 8-9)

Terminology Explanation
Virtual Address An address used by the CPU that must be translated to a physical address
Physical Address The actual address in RAM hardware
MMU (Memory Management Unit) Hardware component that translates virtual addresses to physical addresses
Address Space The range of virtual addresses available to a process
Valid Bit A flag in the page table indicating whether a page is currently in RAM (1) or not (0)
Present Bit Indicates whether a page is currently loaded in physical memory
Protection Bits Flags that control access permissions (read, write, execute) for a page
Page Fault Handler The OS routine that handles page fault exceptions
Trap / Exception A signal to the OS that something requires its attention (like a page fault)
Kernel Mode A privileged CPU mode where the OS has full access to hardware
User Mode A restricted CPU mode where applications run with limited privileges
Context Switch Saving the current process state and switching to another (or the OS)
Interrupt Return (IRET) An instruction that returns from an interrupt handler, restoring the previous CPU state
Program Counter (PC) Register holding the address of the next instruction to execute
Processor Status Register Register containing CPU flags and mode information

Section 9: Memory Protection & Validation (Page 8)

Terminology Explanation
Read-Only Region A memory area where write operations are prohibited
Illegal Access A memory access that violates protection rules (e.g., writing to read-only memory)
Demand-Zero Page A newly allocated page that hasn’t been used yet — the OS provides a fresh page filled with zeros
Mapped File A file whose contents are mapped into a process’s virtual address space
